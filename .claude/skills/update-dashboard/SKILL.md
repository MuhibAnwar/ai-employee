# Update Dashboard

Refresh `vault/Dashboard.md` with the current state of the entire vault.
Gold Tier dashboard includes all 6 watcher statuses, per-domain pending item
counts, and links to the latest Accounting audit and CEO Briefing.

## When to Use

- After processing inbox items
- After triaging needs-action items
- After completing any task
- Any time you want a current status overview

## Steps

1. **Audit all vault folders** — count non-hidden, non-`.gitkeep` files:

   | Folder | Count Variable |
   |--------|----------------|
   | `vault/Inbox/` | `inbox_count` |
   | `vault/Needs_Action/` | `needs_action_count` |
   | `vault/Pending_Approval/` | `pending_approval_count` |
   | `vault/Plans/` | `plans_count` |
   | `vault/Approved/` | `approved_count` |
   | `vault/Done/` | `done_count` |
   | `vault/Accounting/` | `accounting_count` |
   | `vault/Briefings/` | `briefings_count` |

2. **Per-domain pending counts** — scan `vault/Needs_Action/` filenames and group by prefix:

   | Prefix | Domain |
   |--------|--------|
   | `FILE_` | Filesystem |
   | `EMAIL_` | Gmail |
   | `LINKEDIN_` | LinkedIn |
   | `FACEBOOK_` | Facebook |
   | `INSTAGRAM_` | Instagram |
   | `TWITTER_` | Twitter/X |
   | `ALERT_` | System Alert |

3. **Watcher health** — read today's log file and the most recent previous log file.
   For each watcher (FileSystemWatcher, GmailWatcher, LinkedInWatcher,
   FacebookWatcher, InstagramWatcher, TwitterWatcher), determine:
   - Status: HEALTHY (no crashes), DEGRADED (crashes but circuit CLOSED), DOWN (circuit OPEN)
   - Use `watcher_circuit_open` entries as the definitive DOWN signal
   - Use `watcher_started` entries to confirm recovery

4. **Read recent activity** — open today's log file at `vault/Logs/YYYY-MM-DD.json`:
   - Extract the last 5 log entries
   - Format as a bullet list for the "Recent Activity" section

5. **Latest audit and briefing** — find the most recent files matching:
   - `vault/Accounting/Weekly_Audit_*.md`
   - `vault/Briefings/*_Monday_CEO_Briefing.md`

6. **Rewrite `vault/Dashboard.md`** using this template:

   ```markdown
   ---
   last_updated: <ISO timestamp>
   status: active
   tier: Gold
   ---

   # AI Employee Dashboard

   > Auto-updated by Claude Code. Run `/update-dashboard` to refresh.

   ## System Status

   | Item | Value |
   |------|-------|
   | AI Employee | Active |
   | Tier | **Gold** |
   | Last Updated | <YYYY-MM-DD HH:MM UTC> |
   | Vault Health | OK |
   | Gold Phase | Phase N of 4 |

   ## Watcher Health (6 Sources)

   | Watcher | Status | Notes |
   |---------|--------|-------|
   | FileSystemWatcher | ✅ HEALTHY / 🟡 DEGRADED / 🔴 DOWN | <brief note> |
   | GmailWatcher | ... | ... |
   | LinkedInWatcher | ... | ... |
   | FacebookWatcher | ... | ... |
   | InstagramWatcher | ... | ... |
   | TwitterWatcher | ... | ... |

   ## Inbox

   - **Pending items:** <inbox_count>
   <list filenames if > 0, else "_Inbox is empty_">

   ## Needs Action by Domain

   - **Total:** <needs_action_count>
     - Filesystem: <n>
     - Email: <n>
     - LinkedIn: <n>
     - Facebook: <n>
     - Instagram: <n>
     - Twitter/X: <n>
     - System Alerts: <n>

   ## Pending Approvals

   - **Awaiting your review:** <pending_approval_count>
   <list filenames if > 0>

   ## Active Plans

   - **In progress:** <plans_count>
   <list plan filenames if > 0>

   ## Accounting

   - **Latest audit:** <most recent Weekly_Audit_*.md filename or "None yet">

   ## Briefings

   - **Latest briefing:** <most recent *_Monday_CEO_Briefing.md filename or "None yet">
   - **Next briefing:** <next Monday date>

   ## Done This Session

   - **Completed items (total):** <done_count>

   ## Recent Activity

   <last 5 log entries as bullet points, or "_No activity yet._">

   ---

   *Generated by AI Employee v0.3 (Gold Tier) — <ISO timestamp>*
   ```

7. **Log the dashboard update** to `vault/Logs/YYYY-MM-DD.json`:
   ```json
   {
     "timestamp": "<ISO>",
     "action_type": "dashboard_updated",
     "actor": "claude_code",
     "source_file": "vault/Dashboard.md",
     "result": "success",
     "notes": "inbox=<n>, needs_action=<n>, pending=<n>, done=<n>, watchers=6"
   }
   ```

8. **Report to user:**
   ```
   Dashboard updated ✅
   Inbox: <n> | Needs Action: <n> | Pending: <n> | Done: <n>
   Watchers: <healthy>/6 healthy
   ```

## Notes

- This is a **read-only** audit — it only writes to `Dashboard.md` and the log
- Never modify files in other folders during this skill
- If a folder doesn't exist, count it as 0 (do not create it here)
- Watcher health is inferred from logs — if no log data, mark as UNKNOWN
- The orchestrator.py also writes a `## Watcher Health` section; the dashboard
  skill writes a unified version that supersedes it
