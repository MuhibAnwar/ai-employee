# Weekly Business Audit

Generate a structured weekly audit of the AI Employee vault and business health.

## When to Use

- Every Sunday night (manually or via scheduled task)
- Any time a full business health snapshot is needed
- Before generating the Monday CEO Briefing

## Pre-Flight

1. Read `vault/Company_Handbook.md` — confirm rules
2. Compute date range: today minus 7 days (inclusive)
3. Check that `vault/Accounting/` directory exists — if not, note it will be created

## Steps

### Step 1 — Scan vault/Done/ (past 7 days)

List all files in `vault/Done/` (skip `.gitkeep`).

For each file, check the filename prefix timestamp or the file's `completed_at` frontmatter.
Include files modified or created within the past 7 days.

Count and categorise:
- `file_processed` — items from Inbox that were processed
- `plan_completed` — PLAN_*.md files moved to Done
- `email_sent` / `email_drafted` — email actions completed
- `social_post` — post_linkedin / post_facebook / post_instagram / post_twitter actions
- `alert_resolved` — ALERT_*.md files moved to Done
- `other` — anything else

### Step 2 — Scan vault/Needs_Action/ and vault/Pending_Approval/

Count items still open (excluding `.gitkeep`). These are pending items.

### Step 3 — Analyse vault/Logs/ for the past 7 days

List all `YYYY-MM-DD.json` log files in `vault/Logs/` that fall within the past 7 days.

For each log file, count:
- Total log entries
- Entries with `"result": "error"` — error count
- Entries with `"action_type": "watcher_started"` — watcher restarts
- Entries with `"action_type": "watcher_circuit_open"` — circuit breakers tripped
- Entries with `"action_type": "watcher_crashed"` — crashes

Identify which watchers had issues (FileSystemWatcher, GmailWatcher, LinkedInWatcher, FacebookWatcher, InstagramWatcher, TwitterWatcher).

For each watcher, determine:
- Status: `HEALTHY` (no circuit trips) | `DEGRADED` (some crashes, circuit closed) | `DOWN` (circuit OPEN)
- Crash count over the period

### Step 4 — Read vault/Business_Goals.md

Extract:
- Monthly revenue target and current MTD
- Key metrics table (target vs current)
- Active projects and their statuses

### Step 5 — Write vault/Accounting/Weekly_Audit_<date>.md

`<date>` = today's date in YYYY-MM-DD format.

Use this template:

```markdown
---
audit_date: <YYYY-MM-DD>
period: <start_date> to <end_date>
generated_by: claude_code
---

# Weekly Audit — <YYYY-MM-DD>

---

## 4. Flagged Items Needing Attention

List any of the following (if present):
- Watchers with circuit breaker OPEN → needs human intervention
- Pending_Approval items older than 48 hours → may be stale
- Business goals with current < 50% of target → needs review
- Errors appearing in logs more than 3 times → recurring issue

_(Note: Odoo financial integration deferred — disk space. Will be added in future update.)_

## 5. Audit Log Reference

Log files reviewed:
- vault/Logs/<date>.json (N entries, N errors)

---

*Generated by AI Employee — weekly-audit skill*
```

### Step 6 — Log the action

Append to `vault/Logs/<today>.json`:

```json
{
  "timestamp": "<ISO>",
  "action_type": "weekly_audit_generated",
  "actor": "claude_code",
  "source_file": "vault/Accounting/Weekly_Audit_<date>.md",
  "result": "success",
  "notes": "Weekly audit for <period>. Tasks completed: N. Errors flagged: N."
}
```

### Step 7 — Update Dashboard

Run `/update-dashboard` after writing the audit file.

## Output Format

```
Weekly Audit complete.

Period: 2026-02-15 to 2026-02-22
Output: vault/Accounting/Weekly_Audit_2026-02-22.md

Summary:
  ✓ Tasks completed this week: N
  ✓ Open items: N
  ⚠ Watcher alerts: N (see audit for details)
  ✓ Business goals: see audit

Dashboard updated.
```

## Notes

- `vault/Accounting/` must exist before writing (create if missing)
- Never overwrite an existing audit file — if it exists, append a timestamp suffix
- Odoo financial data (invoices, expenses, P&L) will be integrated in a future update when disk space is available
- This audit is the source of truth for the Monday CEO Briefing — run it first
