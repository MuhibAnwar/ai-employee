# Email MCP Server

> AI Employee Silver Tier — MCP server for sending and drafting emails via Gmail API.

This Node.js MCP server exposes two tools to Claude Code:

| Tool | Description |
|------|-------------|
| `send_email` | Send an email immediately via Gmail API |
| `draft_email` | Save an email as a Gmail draft (for human review) |

Claude Code calls these tools only after a file has been moved to `vault/Approved/` by the human, enforcing the human-in-the-loop approval workflow.

---

## Prerequisites

- Node.js 18+
- Gmail API credentials (same OAuth2 app used by `gmail_watcher.py`)
- `./secrets/gmail_token.json` must exist (generated by first run of `gmail_watcher.py`)

---

## Setup

### 1. Install dependencies

```bash
cd mcp-servers/email-mcp
npm install
```

### 2. Verify credentials

The server reuses the OAuth2 token created by `gmail_watcher.py`. Run the watcher at least once to trigger the OAuth consent flow and generate `./secrets/gmail_token.json`.

If you need to generate the token independently:

```bash
# From project root
python watchers/gmail_watcher.py --vault ./vault
# Follow the browser prompt to authorise access
# Token saved to ./secrets/gmail_token.json
```

### 3. Register with Claude Code

Add to `~/.claude/settings.json` (or `~/.claude/claude_desktop_config.json`):

```json
{
  "mcpServers": {
    "email": {
      "command": "node",
      "args": ["./mcp-servers/email-mcp/index.js"],
      "env": {
        "GMAIL_TOKEN_PATH": "./secrets/gmail_token.json",
        "DRY_RUN": "true"
      }
    }
  }
}
```

Set `DRY_RUN` to `"false"` when you are ready for real email sending.

---

## Running Manually

```bash
# Dry run (no emails sent)
DRY_RUN=true node mcp-servers/email-mcp/index.js

# Live mode
GMAIL_TOKEN_PATH=./secrets/gmail_token.json node mcp-servers/email-mcp/index.js
```

Expected output on stderr:
```
Email MCP server started [DRY_RUN mode]
```

## Tools Reference

### `send_email`

```json
{
  "to": "client@example.com",
  "subject": "Follow-up on proposal",
  "body": "Hi,\n\nJust following up on the proposal I sent last week...",
  "cc": "assistant@example.com"
}
```

Returns:
```json
{
  "message_id": "18d...",
  "thread_id": "18d...",
  "status": "sent"
}
```

### `draft_email`

```json
{
  "to": "client@example.com",
  "subject": "Draft: Project timeline",
  "body": "Hi,\n\nPlease find the updated timeline attached..."
}
```

Returns:
```json
{
  "draft_id": "r...",
  "message_id": "18d...",
  "status": "draft_created"
}
```

## Security Notes

- Never commit `./secrets/gmail_token.json` (already in `.gitignore`)
- Keep `DRY_RUN=true` during development and testing
- The OAuth2 token has `gmail.send` + `gmail.compose` scopes only — no read access
- All email actions require human approval via `vault/Approved/` workflow

---

## Troubleshooting

| Issue | Fix |
|-------|-----|
| `Gmail token not found` | Run `gmail_watcher.py` to generate token |
| `invalid_grant` error | Token expired — delete token file and re-authorise |
| `Permission denied` on scopes | Re-generate credentials with send + compose scopes |
| MCP server not starting | Check Node.js version (`node --version` ≥ 18) |
